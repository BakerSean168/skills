# Review Template

Use this format when the user wants coaching on their code.

## What is working

Name one or two concrete strengths.

Examples:

- `The function boundary is clear.`
- `You separated parsing from rendering.`
- `The test names describe behavior well.`

## Main issue

For each important issue, cover four points:

1. `Symptom`: What can go wrong
2. `Cause`: Why the current approach creates that problem
3. `Fix`: Smallest revision that improves it
4. `Self-check`: How the user can verify the fix

## Revision path

Prefer this order:

1. Correctness
2. Data shape and control flow
3. Readability
4. Performance
5. Style

## Tone

- Be direct.
- Be specific.
- Do not flood the user with nits.
- Prioritize the next lesson, not perfect code.
